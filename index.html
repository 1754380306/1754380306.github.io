题目描述

给定一个数字序列 {X1, X2, … , Xn}, 假定 Xk = (A * k + B) % mod。你的任务就是找一个最大的子序列 {Y1, Y2, … , Ym} ，对于每对 (Yi, Yj) 要求满足 Yi + Yj <= L (1 ≤ i < j ≤ m)，而且同时满足 Yi <= L (1 ≤ i ≤ m )。现在给定 n，L，A，B 和 mod，你的任务就是找出上述最大的 m 值。

输入格式

输入包括多组测试数据，直到文件尾。每个测试数据占一行，包括 5 个整数：n、L、A、B 和 mod。其中，n 表示数字序列中元素的个数；L 表示要求的子序列满足的条件；A、B、mod 是用来生成数字序列的三个参数。1 ≤ n ≤ 2 * 10^7, 1 ≤ L ≤ 2 * 10^9, 1 ≤ A, B, mod ≤ 10^9。

输出格式

对于每组测试数据，输出符合要求的最大子序列的元素个数 m，占一行。

解题思路

本题需要寻找具有特殊性质的最大子序列，而生成数列的方式也比较特殊。一种可行的解题思路如下：

  1. 按照公式 (A * k + B) % mod 生成数字序列 {X1, X2, … , Xn}；
  2. 遍历数字序列，统计小于等于 L/2 的元素个数 num，并记录其中最大的值 ma；
  3. 再次遍历数字序列，找到第一个大于 L - ma 且大于 L/2 的元素，将 num 增加 1；
  4. 输出 num，即为符合要求的最大子序列的元素个数。

教育技术人的专业发展

教育技术学的学科性质

基础性学科:研究知识,如数理化文史哲

工具性学科:解决问题

教育技术学是教育学的二级学科主要解决的事与实践紧密相连的教育教学的工具/方法和策略等具体问题

教育技术学是工具性学科.

教育技术学在大教育学科群中的地位是由教育技术在教学过程中所发挥的作用决定的.

目前教育技术学是大教育学科中的显学.

教育技术学是综合交叉学科,杂而不落,”杂”的背后是严密的学科框架和概念体系,具有极强的内在逻辑性.

教育技术学学科基本特征

  • 研究和实践主题的多元化
  • 学习资源的开发与应用
  • 系统方法设计和组织教学过程
  • 以改善和获得有效的学习结果为目标
  • 以解决教育,教学目的为目的

教育技术学研究方法

教育科学研究的层次

  • 基础研究(思想者):多读书,发现新领域,新规律提出新观点,新学说,新理论.(不太适用于教育技术学)
  • 应用研究(学者):运用基础理论研究的成果解决教育工作中的具体问题,是基础研究的具体化和操作化.如调查研究,实验研究(本科阶段)
  • 开发研究(改革家):做进一步推广,扩大影响价值,寻求更明确的具体技术的表现形式(毕业论文阶段)

教育技术学研究的类别

  • 实践研究:教育实验研究/教育调查/课堂观察发/行动研究法/基于设计的研究
  • 评价研究:内容分析法,德尔菲法,解释结构模型法,模糊综合评判,层次分析法
  • 理论研究:

教育实验研究

  • 教育实验的含义:根据一定的理论和假设,通过人为的控制教育现象中某些因素,从而探索变量之间因果关系的研究方法.
  • 研究过程:在科学的教育理论指导下,提出一个具有因果关系的假设,一次为出发点,选取被试,按照某种方式对被试实施实验处理和测量,通过统计分析确定假设是否成立,论证某一因果关系

实验设计的基本要素

  • 自变量:由实验者操纵,掌握的吃即变量,目的是对被试产生影响.自变量可以采用的值叫做自变量的水平.在教育实验中,教学环境/教学方法/教材内容/课程结构/师生关系/被试者个人特点等都可以作为自变量.
  • 因变量:反应被试变化的反映变量,是由自变量刺激作用而导致的结果.在教育实验中,因变量往往是与被试全身心发展的水平/程度/状况有关,如对知识的掌握水平/能力发展/思想品德的发展程度/情感的发展/教学质量等
  • 无关变量:泛指除自变量以外一切可能影响研究结果的因素.无关变量对实验并不是真的”无关”,而只是未被当作自变量,是本次研究不关注的变量.

真实验与准实验

  • 真实验:严格,基本逻辑是根据随机化的原则把被试分配到不同的实验条件中去,所形成的这些组具有同质性或是等组.只要能严格控制无关变量的实验,都可以成为真实验
  • 准实验:介于真实验和非实验之间的一种实验设计,不需要采用随机化的方法来分配被试,但能够严格地操纵自变量和控制无关变量,适合更广泛的研究目的,在难以满足真实验条件下做准实验

协作学习活动

协作学习是通过小组或团队的形式组织学生进行学习的一种策略.

协作学习是学生以小组形式参与,为达到共同的学习目标,在一定的激励机制下最大化个人和他人的成果,而合作互助的一切相关行为

协作学习以小组活动为主体,强调异质分组,强调小组成员的合作互助,强调目标导向,强调总体成绩作为激励.

作为老师在布置小组写作任务时需要注意:

  • 设置课题的难度,避免一个人全干完
  • 客观评鉴

可能出现下面问题:

  • 协作学习中出现搭便车现象
  • 林戈尔曼效应:当人们参加社会集体活动时,个人贡献会因人数的增加而逐渐减少,林格尔曼将其称之为”社会惰性”.

为解决搭便车:

  • 在开始活动时,分配任务
  • 结束后互评

为解决林格尔曼效应:

  • 将小组内再进行分组

协作活动五要素:

  1. 积极互赖:不仅为自己的学习负责,也为小组的其他同伴的学习负责
  2. 个体职能:每个学生都承担一定学习任务,成功取决于所有组员个人的学习
  3. 小组自加工:反思在某一活动时期某些组员的活动有益无益,哪些活动可以继续或者需要改进
  4. 社交技能:必要的社会技能
  5. 面对面的促进性交互:面对面的交流对学习产生直接促进作用

JDBC学习笔记

JDBC（Java数据库连接）是Java编程语言中一个用于规范客户端程序如何访问数据库的应用程序接口，提供了一种与各种关系型数据库交互的方式。本文将介绍JDBC的基础知识和使用方法。

JDBC的基本概念

JDBC主要由两个部分组成：JDBC API 和 JDBC驱动程序。

  • JDBC API是一组Java类和接口，它们定义了Java程序如何与关系型数据库进行通信。JDBC API包括三个层次结构：第一层是管理与数据源的连接，第二层是执行SQL语句，第三层是处理结果集。
  • JDBC驱动程序则是针对不同数据库的具体实现，它们将JDBC API的调用转化为适合特定数据库的协议。根据JDBC规范的不同，JDBC驱动程序可以分为四种类型：JDBC-ODBC桥接驱动、本地API驱动、网络协议驱动和纯Java驱动。

在使用JDBC之前，需要下载并安装相应的JDBC驱动程序。以MySQL为例，我们可以从MySQL官网下载JDBC驱动程序，并将其添加到项目的classpath下。

Servlet+mvc/jar包介绍/如何导入使用

请补充下面文字中讲得比较简略的部分Servlet 是 Java Web 开发中的重要组成部分之一，它是一种规范（一个组织设计的一种标准），用于处理 B/S（浏览器/服务器）结构下的浏览器请求。与底层源码无关，开发者可以在实现 Servlet 时拿出用户名和密码，并从数据库中检索用户信息。

Servlet 规范了许多接口、类、文件内容和路径，其中 Java SE 是标准版，而 Java EE 则是商业版本。Tomcat 是遵循 Servlet 规范的一种服务器。

开发 Servlet 应用程序大致需要经过如下步骤：

在项目目录下新建一个名为 crm 的目录。
新建一个名为 WEB-INF 的目录，在其中创建 classes 和 lib 目录以及一个名为 web.xml 的文件。
在 web.xml 文件中设置 Servlet 的映射关系。
实现 Servlet 类，并在其中重写 init、service 和 destroy 方法。
部署应用程序，并启动 Tomcat 服务器。
Servlet 生命周期包括了如下几个阶段：

在第一次被用户访问时创建，
服务器关闭时销毁，
Servlet 对象只有一个（单例），
对象中的成员变量都是共享的。
Servlet 的维护由服务器控制，生命周期也由 Web 控制。不建议手动定义构造函数。

在 Servlet 的 destroy 方法中，可以将数据存储到应该到的位置，在 init 方法中加载文件。

GenericServlet 是设计程序的理论，是一个抽象类。它解决了直接编写 Servlet 复杂且必须要实现 init 和 destroy 的问题。ServletConfig 存储了配置信息，并存储了 web.xml 配置信息，一一对应。ServletContext 环境（上下文）是 Web 应用中唯一的，生命周期由服务器控制。

HTTP 协议是一种基于 TCP/IP 协议的应用层协议，为 Web 浏览器和 Web 服务器之间的通信设计。HTTPServlet 继承自 GenericServlet 并专门针对 HTTP 协议的请求做了一些调整和优化。

在 HTTP 协议中，客户端发送请求并获取响应。请求包含请求头、请求体和请求行，其中请求类型包括 get、post、delete、put、head、option 和 trace。响应包括状态行、响应头、空白行和响应体。GMT 标准时间表示时区标准。
